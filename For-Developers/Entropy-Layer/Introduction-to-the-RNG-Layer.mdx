While the core Rain protocol is designed for event-based prediction markets that rely on individual liquidity pools and real-world resolutions, the Entropy Layer is a specialized infrastructure component built for high-speed markets and applications.

The Entropy Layer uses verifiable randomness powered by Chainlink VRF to resolve markets instantly.  It provides developers with a framework to build high-velocity, randomly pre-generated scenarios that require immediate outcomes without waiting for external news events to conclude.

## How it Differs from Standard Markets

The most significant difference between the standard Rain SDK and the Entropy Layer is how liquidity is handled. Instead of requiring builders to spin up a new, isolated liquidity pool for every single market, the Entropy Layer shifts the protocol structure to a limited shared liquidity model.

- **Instant Settlement:** Because liquidity is drawn from a limited shared pool, user trades and outcomes can be settled instantly.
- **Liquidity Provisioning:** Users can participate as Liquidity Providers (LPs) by funding the limited shared pool. LPs earn a percentage of the transaction volume generated by all applications running on this layer.
- **Stake Retention:** If participants in a scenario fail to exit their positions before a random termination point is reached, their stakes remain in the pool, directly increasing the value of the LP shares.

## What You Can Build

Because the Entropy Layer abstracts away the complexity of managing individual pools, you can focus entirely on building engaging, high-speed user experiences. Here are a few examples of what you can build on top of the Entropy Layer:

- **Risk Markets:** Build high-speed risk environments (like RiskMarkets.fun) where users must manage profit escalations in real-time before the RNG engine determines the precise moment the scenario ends.
- **Randomly Pre-Generated Scenarios:** Create markets where the outcomes are determined instantly by verifiable randomness rather than real-world judges or AI oracles.
- **High-Speed Trading Interfaces:** Build specialized frontends that allow users to rapidly enter and exit scenarios, utilizing the limited shared liquidity pool for immediate trade execution.

## The Entropy Layer SDK

To make building on this layer as simple as possible, we provide the Entropy Layer SDK (`rng-layer-sdk`). It is a clean, modular TypeScript SDK specifically designed to interact with the protocol programmatically.

The SDK is divided into two main entry-point classes:

1. **Liquidity:** Handles all interactions with the limited shared pool. You can use it to read the pool's state (like total liquidity and max trade sizes) and build transactions for users to deposit, withdraw, or unreserve funds.
2. **Allowance:** A utility class that handles USDT token allowance checks, ensuring smart accounts have the correct permissions before interacting with the pool.
